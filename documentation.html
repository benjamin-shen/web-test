Description

Our website allows a teacher to administer tests online.
Students are able to take the tests and view their scores (questions are shown individually, allowing the student to leave a test incomplete).
The teacher is able to view everyone's scores (and statistics) and manage the class (by registering new students and creating new tests).

Recommended order of testing (no pun intended):

  1. Read rest of documentation
  2. Glimpse at names in Student Login
  3. Login as a teacher using the password 0 and action "View Scores"
  4. Admire the statistics table
  5. Choose the action "Register a Student"
  6. Register a unique person that's not already on the students list
  7. Login in as that student (or another student - Test Students' passwords are 0) and take the sample test
  8. Try to cheat by refreshing the page, pressing the back key, or logging out and in again
  9. Keep track of your score and confirm it at the congrats page
  10. Log in as a teacher to view your score and confirm it, and experiment with the reset button
  11. Add a short test that doesn't already exist and take it
  12. Repeat any part of this list and try to break the program

Bugs

  • pressing the enter key in the login page always attempts the student login (to log in as teacher, click the appropriate button)
  • some features may produce errors if the user is using a browser earlier than Internet Explorer 9 (ex. the number input type isn't supported)
  • writing random query strings in the URL produces errors
  • in test creation, no input can be the same as the test name

Security flaws (self-added)

  • test answers are publicly available (encrypting answers would still produce a/b/c/d patterns)
  • since others have read/write permission, student scores are viewable and potentially changeable
  • a hacker can type ?name=name at the end of the after_login.py URL to automatically log in without a password
